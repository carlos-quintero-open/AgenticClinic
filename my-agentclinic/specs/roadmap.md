# AgentClinic Roadmap

## Implementation Strategy: Nano-Phases

Each phase contains 1-3 small features, completable in a day or less. This enables rapid iteration, quick feedback cycles, and continuous demo-ability.

## Phase 1: Foundation
- [ ] Project setup (TypeScript, framework, SQLite)
- [ ] Basic server running

## Phase 2: Core Data Models
- [ ] Agent entity (name, status, stress level)
- [ ] Ailment/Therapy concept

## Phase 3: Basic API
- [ ] GET agents endpoint
- [ ] GET single agent endpoint

## Phase 4: Create Operations
- [ ] POST new agent
- [ ] POST ailment report

## Phase 5: Dashboard Shell
- [ ] Basic UI to display agents
- [ ] Simple HTML/CSS structure

## Phase 6: Dashboard Features
- [ ] View agent details
- [ ] View ailments per agent

## Phase 7: Booking System
- [ ] Therapy appointment model
- [ ] Schedule appointment endpoint

## Phase 8: Therapy & Treatment
- [ ] Therapy types/options
- [ ] Apply therapy to agent

## Phase 9: UI Enhancements
- [ ] Playful, satirical styling
- [ ] Animation/UX polish

## Phase 10: Polish & Demo
- [ ] Bug fixes
- [ ] Demo readiness
- [ ] Documentation

---

*This roadmap is intentionally modular for conference booth demos, classroom teaching, and iterative delivery.*
